

# Prompt 6D: Request for Quote (RFQ) Flow + Cart Button Wiring

## Overview
Create the RFQ page, wire cart buttons, and make "Request Quote" text clickable on ProductCard/ProductDetail. Total: 1 new file, 3 modified files, 1 route addition.

## 1. New File: `src/pages/RequestQuotePage.tsx`

- Route: `/request-quote`
- **Entry points**:
  - `?from=cart` → auto-load cart items (query cart_items + products_public, same pattern as CartPage)
  - `?product={id}` → load single product
  - Direct visit → empty form for manual entry
- **Form fields**:
  - Project Type dropdown (Kitchen Setup, Restaurant Renovation, Hotel Fit-Out, Catering Event, Regular Supply, Other)
  - Timeline dropdown (Urgent 1 week, Standard 2-4 weeks, Flexible 1-3 months)
  - Budget Range dropdown (Under 5M, 5-20M, 20-50M, 50-100M, Over 100M MMK)
  - Items table: product name, quantity, notes per item; "Add Row" / "Remove" buttons
  - Additional Notes textarea
  - "Submit Quote Request" button (navy)
- **On submit**:
  - Get `customer_id` via `get_customer_id_for_user` RPC
  - Insert into `quotes` table: `customer_id`, `status='pending'`, `items` as JSONB, `project_type`, `timeline`, `budget_range`, `additional_notes`
  - Quote number auto-generated by existing `generate_quote_number` trigger
  - Fetch the inserted quote to get `quote_number` for success message
  - If `from=cart`: delete all cart_items for customer, invalidate cart queries
  - Show success message with quote number
  - Navigate to `/account` (quotes section)
- **Auth guard**: redirect to home or open auth modal if not logged in
